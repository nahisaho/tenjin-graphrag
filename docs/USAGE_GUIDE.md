## 基本的な使い方

### 理論を検索する

**キーワード検索:**
```
「認知負荷」に関連する教育理論を検索してください
```

**カテゴリ検索:**
```
学習理論（learningカテゴリ）を一覧表示してください
```

**エビデンスレベルでフィルタ:**
```
エビデンスレベルがstrongの理論を検索してください
```

### 理論の詳細を取得する

```
認知負荷理論（cognitive-load-theory）の詳細を教えてください
```

**出力される情報:**
- 理論の説明
- 提唱者
- 関連する概念
- 主要な原則
- 支持するエビデンス

### 理論家について調べる

```
ジョン・スウェラー（sweller）について教えてください
```

## ユースケース別ガイド

### 1. 授業設計

#### シナリオ: 小学校算数の授業を設計する

**Step 1: 関連理論を探す**
```
小学校の算数授業に適した教育理論を検索してください。
学習段階を考慮した理論を優先してください。
```

**Step 2: 理論の詳細を確認**
```
スキャフォールディングの原則を詳しく教えてください
```

**Step 3: 授業設計プロンプトを使用**
```
「分数の概念」をテーマに、小学4年生を対象とした授業を設計してください。
足場かけ理論とマルチメディア学習理論を適用してください。
```

#### 期待される出力:
- 学習目標の設定
- 理論に基づいた教授戦略
- 段階的なスキャフォールディング計画
- 認知負荷を考慮した教材設計
- 評価方法の提案

---

### 2. 研究・文献調査

#### シナリオ: 論文執筆のための文献調査

**Step 1: 関連理論の探索**
```
自己調整学習に関連する理論を検索し、それぞれの関係性を示してください
```

**Step 2: 理論間の関係をトラバース**
```
自己決定理論から関連する理論をたどってください（深さ2）
```

**Step 3: 引用文献を生成**
```
以下の理論についてAPA7形式の引用を生成してください:
- cognitive-load-theory
- self-determination-theory
- zone-of-proximal-development
```

#### 期待される出力:
```
Sweller, J. (1988). Cognitive load during problem solving: 
Effects on learning. Cognitive Science, 12(2), 257-285.

Ryan, R. M., & Deci, E. L. (2000). Self-determination theory 
and the facilitation of intrinsic motivation, social development, 
and well-being. American Psychologist, 55(1), 68-78.

Vygotsky, L. S. (1978). Mind in society: The development of 
higher psychological processes. Harvard University Press.
```

### 3. 教授法選定

#### シナリオ: オンライン授業に最適な教授法を選ぶ

**Step 1: 教育文脈を検索**
```
高等教育のオンライン環境に適した教育文脈を検索してください
```

**Step 2: 教授法を推薦**
```
以下の条件で教授法を推薦してください:
- 文脈: higher-education-online
- エビデンスレベル: moderate以上
```

**Step 3: 推薦された教授法の詳細を確認**
```
反転授業（flipped-learning）の詳細と理論的根拠を教えてください
```

#### 推薦の判断基準:
1. **理論的根拠**: どの理論に基づいているか
2. **エビデンスレベル**: 実証研究の強さ
3. **適用文脈**: 対象とする学習者・環境との適合性
4. **実践的考慮**: 実装の難易度とリソース要件

---

### 4. 理論比較分析

#### シナリオ: 学習理論のアプローチを比較する

**Step 1: 比較する理論を選択**
```
行動主義、認知主義、構成主義を比較してください
```

**Step 2: 詳細な比較分析**
```
以下の観点で比較分析してください:
- 学習の定義
- 教師の役割
- 学習者の役割
- 評価方法
- 適用場面
```

**Step 3: 関係性の可視化**
```
これらの理論間の関係（BUILDS_ON, CONTRASTS_WITH, COMPLEMENTS）を
説明してください
```

#### 比較出力例:

| 観点 | 行動主義 | 認知主義 | 構成主義 |
|-----|---------|---------|---------|
| 学習の定義 | 行動の変化 | 知識の獲得・処理 | 意味の構成 |
| 教師の役割 | 強化の提供者 | 情報の組織者 | ファシリテーター |
| 学習者の役割 | 受動的 | 情報処理者 | 能動的構成者 |
| 主な方法 | ドリル、強化 | 教授設計、チャンク | 協調学習、PBL |

## 高度な使い方

### グラフトラバーサルの活用

**複数ホップのトラバース:**
```
認知負荷理論から3ホップ以内で到達できる全ての概念と原則を表示してください
```

**特定の関係タイプに絞る:**
```
認知負荷理論に関連する理論のうち、BUILDS_ON関係にあるものだけを表示してください
```

**パス探索:**
```
行動主義（behaviorism）から構成主義（constructivism）への
理論的な発展パスを示してください
```

### セマンティック検索の活用

```
「生徒の主体性を高める」教育アプローチに関連する理論を検索してください
```

この検索では、キーワード一致だけでなく、意味的に関連する理論も見つかります。

### 複合クエリ

```
以下の条件を満たす教育実践のアプローチを提案してください:
1. 対象: 大学1年生（初年次教育）
2. 科目: プログラミング入門
3. 環境: ハイブリッド（対面+オンライン）
4. 目標: 問題解決能力の育成
5. 制約: 30人クラス、週1回90分

関連する理論と教授法を理論的根拠とともに推薦してください
```

## トラブルシューティング

### よくある問題と解決策

#### 「理論が見つかりません」エラー

**原因:**
- IDのスペルミス
- 英語/日本語の混同

**解決策:**
```
# まず理論一覧を確認
theory://list リソースで全理論を確認してください
```

#### 検索結果が空

**原因:**
- フィルタ条件が厳しすぎる
- 該当データがない

**解決策:**
1. カテゴリやエビデンスレベルのフィルタを外す
2. より一般的なキーワードで検索
3. `get_graph_statistics` でデータ状況を確認

#### グラフトラバーサルで結果が少ない

**原因:**
- `max_depth` が小さい
- 関係タイプが限定的

**解決策:**
```
# 深度を増やす
max_depth: 3 または 4 に設定

# 全ての関係タイプを対象にする
relationship_types: null（指定なし）
```

### データの確認方法

**グラフ統計を確認:**
```
グラフの統計情報を表示してください（ノード数、関係数など）
```

**スキーマを確認:**
```
graph://schema リソースでグラフのスキーマを確認してください
```

## 付録: ID命名規則

### 理論ID
- 形式: `kebab-case`（小文字、ハイフン区切り）
- 例: `cognitive-load-theory`, `zone-of-proximal-development`

### 理論家ID
- 形式: 姓のみ（小文字）
- 例: `sweller`, `vygotsky`, `piaget`

### 概念ID
- 形式: `kebab-case`
- 例: `intrinsic-load`, `scaffolding`, `prior-knowledge`

### 教授法ID
- 形式: `kebab-case`
- 例: `direct-instruction`, `problem-based-learning`, `flipped-learning`

### 文脈ID
- 形式: `education-level-subject-environment`
- 例: `k12-stem-classroom`, `higher-education-online`
